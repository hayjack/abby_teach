# 教学Web系统设计文档

个人学习使用

## 1. 系统架构

### 1.1 技术栈

- 后端：Python 3 + Flask
- 前端：Vue
- 数据库：PostgreSQL

### 1.2 系统模块划分

| 模块名称 | 主要功能                      | 子模块                 |
| ---- | ------------------------- | ------------------- |
| 系统管理 | 教师信息管理、角色菜单管理             | 教师管理、角色管理、菜单管理、权限管理 |
| 学生管理 | 学生信息管理、家长信息管理、缴费记录管理、课时管理 | 学生信息、家长信息、缴费记录、课时管理 |
| 班级管理 | 班级信息管理、学生班级关联、教师班级关联      | 班级信息、学生班级关联、教师班级关联  |
| 课程管理 | 课程信息管理、总课时设置、班级课程关联       | 课程信息、总课时设置、班级课程关联   |
| 上课记录 | 上课记录录入、自动扣除课时             | 上课记录录入、课时扣除         |
| 请假记录 | 请假记录管理、不扣除课时              | 请假记录录入、请假状态管理       |
| 报表统计 | 学生上课统计、请假统计、教师上课统计        | 学生上课统计、请假统计、教师上课统计  |

## 4. 系统流程设计

### 4.1 上课流程

1. 教师登录系统
2. 进入上课记录模块
3. 选择班级和课程
4. 录入上课日期、时间、内容等信息
5. 系统自动生成考勤记录
6. 教师确认考勤状态（出勤/请假/旷课）
7. 系统自动扣除出勤学生的课时
8. 保存上课记录

### 4.2 请假流程

1. 家长或教师提交请假申请
2. 管理员或教师审批请假申请
3. 请假批准后，系统标记学生请假状态
4. 上课后，请假学生不扣除课时

### 4.3 课时管理流程

1. 学生缴费后，为学生增加课时
2. 学生上课后，系统自动扣除课时
3. 系统定期提醒课时不足的学生

## 6. 技术实现方案

### 6.1 后端实现

- 使用Flask框架搭建RESTful API
- 使用SQLAlchemy ORM操作数据库
- 使用JWT进行身份认证
- 使用Redis进行缓存

### 6.2 前端实现

- 使用Vue 3 + Vite搭建前端项目
- 使用Element Plus作为UI框架
- 使用Vue Router进行路由管理
- 使用Pinia进行状态管理
- 使用Axios进行API调用

### 6.3 数据库实现

- 使用PostgreSQL作为数据库
- 使用SQLAlchemy进行数据库操作
- 数据库表结构按照设计文档创建
